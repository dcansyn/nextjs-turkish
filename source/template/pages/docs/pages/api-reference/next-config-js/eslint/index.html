eslint

Projenizde ESLint algılandığında, Next.js hatalar mevcut olduğunda üretim derlemenizde başarısız olur (next build).

Next.js'nin uygulamanızda ESLint hataları olsa bile üretim kodu üretmesini istiyorsanız, yerleşik linting adımını tamamen devre dışı bırakabilirsiniz. ESLint'i iş akışınızın ayrı bir bölümünde (örneğin, CI'da veya bir ön taahhüt kancasında) çalışacak şekilde yapılandırmadığınız sürece bu önerilmez.

next.config.js adresini açın ve eslint yapılandırmasında ignoreDuringBuilds seçeneğini etkinleştirin:

next.config.js
module.exports = {
  eslint: {
    // Warning: This allows production builds to successfully complete even if
    // your project has ESLint errors.
    ignoreDuringBuilds: true,
  },
}
